# 引用詢問互動修正設計

## 目標

修正桌面版引用詢問的箭頭外觀、浮動按鈕互動色、模型回答語氣與來源定位回饋。

## 核准行為

1. 輸入欄與送出後引用共用同一種短箭頭：直段較短、轉彎使用可見圓弧、線端與接角皆為圓角，不再使用字型字元 `↳`。
2. 浮動「引用詢問」按鈕平時為黑色文字與白色背景；滑鼠 hover 或鍵盤 focus 時文字才變成主題藍色，背景始終維持白色。
3. 引用文字只作為隱藏上下文。模型必須直接回答，不得以「根據引用內容」、「依照上述內容」或其他描述回答依據的句子開場。英文、繁體中文、法文提示同步更新。
4. 點擊已送出的引用後，畫面捲動至原始 AI 文字，該文字以瀏覽器原生選取色反藍 1.2 秒後取消。如果使用者在倒數期間建立了新的文字選取，不得清除使用者的新選取。
5. 保留桌面限定、最多三行、點擊來源定位與既有傳送資料格式。

## 技術選擇

- 箭頭使用 inline SVG 與二次曲線/圓弧路徑，避免不同系統字型造成形狀差異。
- 暫時反藍使用 Range 與 Selection API，因為需求就是原生文字選取色；清除前比對目前 Range，避免誤清除新選取。
- 不修改訊息儲存、雲端同步或提交生命週期。

## 驗證

- 先新增箭頭、顏色、提示與暫時 Range 選取的失敗測試。
- 聚焦測試轉綠後執行完整測試、正式建置與 `git diff --check`。
- 不啟動 localhost 或本機瀏覽器測試。
